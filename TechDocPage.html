What is GitHub? An Introduction

According to w3schools, GitHub is, "a code hosting platform for collaboration and version control. Github lets you and others work together on projects." Basically, it is a site where computer coders can store, share, and collaborate on coding projects.

Why is GitHub Important?

GitHub is important for several reasons. Some reasons are:

  1. It makes it easy for you and your partners to contribute to and manage coding projects.
  2. It showcases your work, which makes it easy for companies looking to hire to see your skills
  3. It allows you to easily track any changes made to your projects.
First Steps

The first thing you should do is go to GitHub.com and create a profile for yourself.

The next step is to, make sure your code files are saved in a folder by themselves and not with all other files. Example: Folder name: CodingProjects (it's better if the folder is one name as spaces are encoded) Folder: TechDocument. If you are not careful, it is possible to upload all files from your C drive onto GitHub which could leave you open to hacking.

Finally, you will need Git Bash installed on your computer. (Windows Users)

Setting up Github

If this is your first time working with Git Hub, you will need to configure it.

In Git Bash configure your name and email by typing these commands:
$ git config --global user.name "Your Name"
$ git config --global user.email youremail.com

Using the Command Line

For the purpose of this document, we will be uploading projects using the Command Line.
The first step in using the Command Line is to launch Git Bash.

Next, we want to make sure we are uploading from the exact project location so we are going to (Change Directory) cd into it. You should already have your file/files stored in a separate folder unique to your project, for example: cd Documents/Projects/ProjectName.

Next, type in ls to list all files in that directory. This way you can also make sure you are not sending sensitive information or the wrong file to GitHub.

Creating a Repository

Before we go any further, make sure you create a repository on GitHub.

  1. Under "Repository" heading, click on "New".

  2. Name your document (do not include spaces. You can put a - in between two words instead of a space.)

  3. Write a description (not necessary but helpful to others looking at your work)

  4. Hit "Create Repository"

That's it, you've created your repository. Next, let's upload your document.

Document Uploading

Now that you have your Repository set up, your Command Line open, and you have changed directories into your project, you need to upload.

  1. Initialize the local directory as a Git repository. $ git init

  2. Add the files in your repository. $ git add.
    (the . after the add will add all files from that directory. If you only want certain files you can type ls to see which files you would like to add, then type in those files).

  3. Now you are ready to commit the files you have staged. $ git commit -m "First commit"
    *This commits the changes and prepares them to be pushed into your repository.

    4. Next, go back to your GitHub project repository's Quick Set Up page. Click the clipboard in the section that says ...or push an existing repository from the command line


  4. Double check that the files you want are included.
    $ git status
    (Files ready are in green.)
  5. Push the file to your local repository.
    $ git push (right click paste)
  6. Go to your GitHub and check that your files have been uploaded.
